# 品牌域名与仓库迁移

## 这块负责什么

这一块不是简单替换站点标题，而是统一替换：

- 产品名
- 公开域名
- GitHub 仓库地址
- 支持邮箱
- Logo / favicon / 分享图
- canonical host 与公开页面 SEO token

## 真相源

- `package.json`
- `src/shared/lib/brand.ts`

- `src/shared/lib/site-icons.ts`
- `src/config/website/public-page-metadata.ts`
- `src/shared/lib/site-discovery.ts`
- `README.md`
- `readurl.md`
- `public/`

## 先搜这些旧 token

```bash
rg -n "Image Editor AI|imageeditorai\.net|support@imageeditorai\.net|CharlieLZ/imageeditorai\.net" .
```

重点检查四类残留：

1. 文案残留
2. URL / canonical 残留
3. support mailto 残留
4. repo URL / Open Graph / schema 残留

## 标准迁移步骤

### 1. 先定一张品牌映射表

至少写清楚这些值：

- `new_app_name`
- `new_app_url`
- `new_domain`
- `new_support_email`
- `new_repository_url`
- `new_theme_color`
- `new_share_image_path`

### 2. 改代码里的 canonical brand token

优先改 `src/shared/lib/brand.ts`，因为它会影响：

- 页面标题
- 文案 token 替换
- repository URL
- support email
- app domain

### 3. 改视觉资源

优先替换：

- logo
- brand mark
- favicon 套件
- Open Graph 分享图

如果只是替换文件但没改 `site-icons.ts` / `site-visuals.ts` 的引用路径，后面很容易出现 manifest 指向旧资源。

### 4. 改公开文案与帮助链接

检查：

- README
- 法律页文案
- landing copy
- pricing copy
- FAQ / mission / support copy

### 5. 重新扫一遍残留

```bash
rg -n "Image Editor AI|imageeditorai\.net|support@imageeditorai\.net|CharlieLZ/imageeditorai\.net" src public README.md readurl.md docs package.json
```

## 验收

至少验证这几件事：

- 首页、Pricing、AI 页面 title / description 都变成新品牌
- `sitemap.xml`、`robots.txt`、`llm*.txt` 不再输出旧域名
- manifest、favicon、Open Graph 图都来自新资源
- support 链接和 repository 链接全部指向新地址

## 常见坑

- 只改 `package.json`，没改 `brand.ts`
- 只改文案，没改 canonical host 集合
- 只换 logo 文件，没换分享图和 manifest
- 忘记把 `imageeditorai_net` 这类数据库 schema 名一起审视，导致“新品牌 + 旧库名”混搭
