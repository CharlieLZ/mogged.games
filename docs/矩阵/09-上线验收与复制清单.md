# 上线验收与复制清单

## 建议把每次新项目都当成固定流水线

顺序不要乱：

1. 复制仓库
2. 换品牌与域名
3. 整理 env 与 Cloudflare
4. 确认数据库、认证、支付、AI、存储
5. 处理 SEO、内容、多语言
6. 跑验证
7. 接 GitHub 自动发布

## 复制清单

### 仓库层

- 新 GitHub 仓库已创建
- `package.json` 的 `name` `homepage` `repository.url` 已更新
- README、readurl、迁移文档都改成新项目名

### 品牌层

- `src/shared/lib/brand.ts` 已更新
- Logo / favicon / share image 已替换
- support email 已切换
- 旧品牌 grep 扫描已清零或只剩注释说明

### 配置层

- `pnpm verify:env` 无 error
- Cloudflare vars / secrets 已配置
- `AUTH_URL`、`NEXT_PUBLIC_APP_URL`、支付回调、邮件回跳一致
- `wrangler.jsonc` 已包含稳定 plain-text Worker vars，不只依赖本地 `.env`
- `scripts/setup-cf-vars.mjs` / `scripts/setup-cf-secrets.sh` 已覆盖新增 key
- `INITIAL_CREDITS_ENABLED`、`INITIAL_CREDITS_AMOUNT`、`INITIAL_CREDITS_VALID_DAYS` 成对确认
- `ADMIN_NOTIFICATION_EMAIL`、`CONTACT_NOTIFICATION_EMAIL`、public support email 已确认收件人职责

### 数据与业务层

- 数据库 schema 已确认
- 登录、注册、支付、上传、生成、查询、任务历史正常
- 注册赠送积分到账，且重复 OAuth hook 不会重复发放
- 登录后浏览器 guest 免费额度仍可用，且和账号 credits 分开
- support/contact 表单能发送或至少能走明确 fallback
- 新项目不需要的后台/证书/旧入口已关掉或删除

### SEO 层

- sitemap 正确
- robots 正确
- llm 路由正确
- locales 与文案状态一致
- `readurl.md` 与实际代码一致

## 推荐验证命令

```bash
pnpm lint:strict
pnpm typecheck
pnpm test:unit
pnpm build
pnpm cf:build
```

仓库内也可以直接跑总 gate：

```bash
pnpm release:gate
```

如果本地需要手动预览 Cloudflare：

```bash
pnpm cf:preview
```

## 发布前人工检查

- 打开首页、Pricing、AI Video、AI Image、Mission、登录页
- 走一遍上传与生成
- 走一遍支付
- 打开 sitemap、robots、llm.txt
- 检查 favicon、分享图、品牌名、footer 链接

## 发布后检查

- Cloudflare 自动发布成功
- 生产域名正确解析
- Webhook 与 Auth 回调正常
- 没有请求回到旧域名
- Worker version 是刚推送的 commit 构建出来的版本
- Worker bindings 里关键 vars / secrets 完整
- `/api/health` 返回 200
- 注册/登录/guest quota/contact/支付 webhook 至少抽查一条

## 不要做的事

- 不要把 `.env` 类文件提交进仓库
- 不要把“仓库没有 GitHub Actions”误写成“没有自动发布”
- 不要为了新项目重新引入旧博客或旧图片优先公开链路
- 不要只用 `cf:vars:sync` 临时修线上，却忘了更新 `wrangler.jsonc`
- 不要把本地能跑当成 Worker runtime 一定能读到 key
