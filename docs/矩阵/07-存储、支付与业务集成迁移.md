# 存储、支付与业务集成迁移

## 这块负责什么

这里不是一个模块，而是一组和“可上线业务”直接相关的集成：

- 存储
- 支付
- 邮件
- 通知
- 客服
- 分析
- 广告
- 联盟营销

## 真相源

- 存储：`src/shared/services/storage.ts` `src/extensions/storage/*`
- 支付：`src/extensions/payment/*` `src/shared/services/payment.ts`
- 邮件：`src/extensions/email/*` `src/shared/services/*email*.ts`
- 通知：`src/extensions/notification/*`
- support/contact：`src/shared/services/support-contact.ts` `src/shared/lib/admin-notification.ts`
- 客服：`src/extensions/customer-service/*`
- 分析：`src/extensions/analytics/*`
- 广告：`src/extensions/ads/*`
- 联盟：`src/extensions/affiliate/*`

## 存储迁移

当前支持：

- R2
- S3

优先检查：

- 上传
- presign
- 完成回调
- 文件回放 URL
- 公共域名 / fallback `/api/storage/file`

必看文件：

- `src/shared/services/storage.ts`
- `src/app/api/storage/*`
- `src/shared/lib/storage-upload-claim.ts`

## 支付迁移

当前支持：

- Stripe
- Creem
- PayPal

标准做法：

1. 先决定只保留哪一个默认支付商。
2. 再检查 `DEFAULT_PAYMENT_PROVIDER`、`STRIPE_ENABLED`、商品映射、Webhook、checkout 页面文案。
3. 最后验证回调、支付状态查询、用户 credits 发放。

必看文件：

- `src/extensions/payment/*`
- `src/app/api/payment/*`
- `src/shared/services/payment.ts`
- `src/shared/lib/payment-config.ts`

## 邮件与通知迁移

邮件 provider 当前主要看：

- Resend
- ZeptoMail

通知看：

- Feishu
- 用户站内通知
- AI 任务完成提醒

如果新项目不需要其中任何一个，应该显式关闭，而不是保留占位配置。

support/contact 收件人要单独复查：

- `CONTACT_NOTIFICATION_EMAIL`
- `ADMIN_NOTIFICATION_EMAIL`
- public/default support email
- 邮件 provider 是否真的能发送

contact 表单的兜底链路是 contact -> admin -> support email。`NEXT_PUBLIC_EMAIL` 只是公开展示和最终兜底，不代表通知收件人已经正确配置。

管理员、支付风控、support/contact 不建议共用一个“看起来像 support 的展示邮箱”糊弄过去。至少要能解释：谁收 payment risk，谁收 contact，谁收 admin report。

## 分析、客服、广告、联盟

这些扩展都适合按需启用，不适合默认全开。

建议每个新项目都先回答：

1. 有没有真实分析需求？
2. 客服是否要在首发当天上线？
3. 联盟计划是否真的准备好了？
4. 广告追踪是否和支付事件闭环一致？

## 验收

- 上传图片/媒体、回放、删除都正常
- checkout、payment callback、status check 正常
- 邮件模板和发件域名正确
- support/contact modal 能发送，失败时有可定位日志，不吞异常
- 分析脚本只在需要时注入
- 客服和联盟脚本没有把旧品牌残留带过来

## 常见坑

- `r2_domain` 没配好，导致媒体 URL 指回旧域名或回退链路
- 开了多个支付商，但默认支付商和前端商品映射不一致
- 新项目不需要某个分析脚本，却把旧 tracking id 带上线
- `CONTACT_NOTIFICATION_EMAIL` 没配，又没有确认 admin/support fallback，导致 contact modal 报 “Could not send”
- 只在本地 `.env` 配了 Stripe / email / analytics key，Cloudflare Worker runtime 没配完整
